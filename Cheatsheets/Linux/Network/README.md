
## Key Networking Topics

| **Topic**      | **Description**                                                                 |
|----------------|---------------------------------------------------------------------------------|
| **DHCP**       | **Dynamic Host Configuration Protocol** automatically assigns IP addresses to devices on a network, simplifying network management. |
| **NAT**        | **Network Address Translation** allows multiple devices on a local network to share a single public IP address, enhancing security and conserving IP addresses. |
| **Firewall**   | A **firewall** is a security device that monitors and controls incoming and outgoing network traffic based on predetermined security rules. |
| **Subnetting** | **Subnetting** divides a larger network into smaller, manageable sub-networks, improving performance and security. |
| **VLAN**       | A **Virtual Local Area Network** (VLAN) segments a physical network into multiple logical networks, enhancing security and reducing broadcast traffic. |
| **VPN**        | A **Virtual Private Network** creates a secure connection over the internet, allowing remote users to access a private network safely. |
| **TCP/IP**     | **Transmission Control Protocol/Internet Protocol** is the fundamental suite of protocols that governs how data is transmitted over the internet. |
| **OSI Model**  | The **Open Systems Interconnection** model is a conceptual framework used to understand and implement network protocols in seven layers. |
| **Routing**    | **Routing** is the process of selecting paths in a network along which to send data packets, often managed by routers. |
| **Switching**  | **Switching** refers to the process of directing data packets between devices on the same network, typically performed by network switches. |
| **DNS**            | **Domain Name System** translates human-readable domain names (like www.example.com) into IP addresses, enabling users to access websites easily. |
| **HTTP/HTTPS**     | **Hypertext Transfer Protocol** (HTTP) is the foundation of data communication on the web, while **HTTPS** adds a layer of security through encryption. |
| **ICMP**           | **Internet Control Message Protocol** is used for error messages and operational information exchange, such as pinging a device to check connectivity. |
| **Load Balancing** | **Load balancing** distributes network or application traffic across multiple servers to ensure reliability and performance. |
| **Proxy Server**   | A **proxy server** acts as an intermediary between a client and the internet, providing anonymity, security, and caching services. |
| **WLAN**           | A **Wireless Local Area Network** allows devices to connect to a network wirelessly, typically using Wi-Fi technology. |
| **QoS**           | **Quality of Service** refers to the management of data traffic to ensure a certain level of performance, prioritizing critical applications. |
| **RIP**            | **Routing Information Protocol** is a distance-vector routing protocol used to determine the best path for data through a network. |
| **BGP**            | **Border Gateway Protocol** is a standardized exterior gateway protocol used to exchange routing information between different autonomous systems on the internet. |
| **SSH**            | **Secure Shell** is a protocol for securely accessing and managing network devices and servers over an unsecured network. |
| **SNMP**           | **Simple Network Management Protocol** is used for network management and monitoring, allowing administrators to manage network devices and gather performance data. |
| **ARP**            | **Address Resolution Protocol** maps IP addresses to MAC addresses, enabling devices on a local network to communicate with each other. |
| **SIP**            | **Session Initiation Protocol** is used for initiating, maintaining, and terminating real-time communication sessions, such as voice and video calls. |
| **IPv4/IPv6**     | **Internet Protocol version 4** (IPv4) and **version 6** (IPv6) are protocols for addressing and routing packets of data across networks, with IPv6 designed to replace IPv4 due to address exhaustion. |
| **Network Topology** | **Network topology** refers to the arrangement of different elements (links, nodes, etc.) in a computer network, including star, ring, mesh, and bus topologies. |
| **Network Segmentation**  | **Network segmentation** involves dividing a network into smaller, isolated segments to improve performance and security. |
| **Access Control List (ACL)** | An **ACL** is a set of rules that controls incoming and outgoing traffic on a network, determining which users or systems can access specific resources. |
| **Network Addressing**    | **Network addressing** refers to the method of assigning unique identifiers (IP addresses) to devices on a network for communication. |
| **Multicast**            | **Multicast** is a method of sending data to multiple recipients simultaneously, optimizing bandwidth usage compared to unicast (one-to-one) or broadcast (one-to-all) methods. |
| **Point-to-Point Protocol (PPP)** | **PPP** is a data link layer protocol used to establish a direct connection between two network nodes, commonly used in dial-up connections. |
| **Frame Relay**          | **Frame Relay** is a high-performance WAN protocol that operates at the data link layer, used for connecting local area networks (LANs) over a shared network. |
| **ATM (Asynchronous Transfer Mode)** | **ATM** is a networking technology that transmits data in fixed-size packets called cells, suitable for real-time applications like voice and video. |
| **Network Time Protocol (NTP)** | **NTP** synchronizes the clocks of computers over a network, ensuring accurate timekeeping for various applications. |
| **SIP Trunking**         | **Session Initiation Protocol (SIP) trunking** allows businesses to connect their private branch exchange (PBX) systems to the internet for voice communication. |
| **Network Function Virtualization (NFV)** | **NFV** decouples network functions from hardware, allowing them to run on virtual machines, enhancing flexibility and scalability. |
| **Content Delivery Network (CDN)** | A **CDN** is a network of distributed servers that deliver web content to users based on their geographic location, improving load times and reliability. |
| **Zero Trust Security**   | **Zero Trust Security** is a security model that assumes no user or device is trustworthy by default, requiring verification for every access request. |
| **Network Policy Server (NPS)** | An **NPS** is a Microsoft server role that provides centralized authentication, authorization, and accounting for network access. |
| **Dynamic Routing Protocols** | **Dynamic routing protocols** (like OSPF, EIGRP, and BGP) automatically adjust the paths that data packets take through a network based on current conditions. |
| **Static Routing**       | **Static routing** involves manually configuring routes in a network, providing more control but requiring more maintenance compared to dynamic routing. |
| **Network Security Groups (NSG)** | **NSGs** are used in cloud environments to control inbound and outbound traffic to resources, enhancing security by defining access rules. |
| **PPTP (Point-to-Point Tunneling Protocol)** | A widely used tunneling protocol that allows for the secure transmission of data over a public network, often used in VPNs, but considered less secure compared to newer protocols. |
| **L2TP (Layer 2 Tunneling Protocol)** | A tunneling protocol that combines features of PPTP and L2F (Layer 2 Forwarding), providing a secure method for transmitting data but typically requires IPsec for encryption. |
| **IPsec (Internet Protocol Security)** | A suite of protocols used to secure Internet Protocol (IP) communications by authenticating and encrypting each IP packet in a communication session. |
| **OpenVPN**                | An open-source VPN protocol that uses SSL/TLS for secure data transmission, known for its flexibility and strong security features. |
| **SSTP (Secure Socket Tunneling Protocol)** | A tunneling protocol that uses SSL/TLS to provide a secure connection over HTTPS, making it suitable for bypassing firewalls. |
| **GRE (Generic Routing Encapsulation)** | A tunneling protocol that encapsulates a wide variety of network layer protocols in a point-to-point connection, often used in conjunction with IPsec for security. |
| **IKEv2 (Internet Key Exchange version 2)** | A tunneling protocol that is part of the IPsec suite, providing secure key exchange and is known for its stability and ability to reconnect quickly after a connection drop. |
| **WireGuard**              | A modern, lightweight VPN protocol that aims to be simpler and faster than existing protocols while providing strong security features. |
